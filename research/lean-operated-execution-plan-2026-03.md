## 2) Success Definition

1. `floor_coverage_ratio >= 1.05`
2. `day-90 retention >= 85%`
3. `intro-to-activation >= 25%`

Pricing-positioning requirements:

1. `undercut_pass = TRUE` against segment benchmark.
2. `coop_member_value_pass = TRUE` (patronage value return + sustainable floor).

Regional launch requirements:

1. Local payment rails live for target region before full launch.
2. Regional scenario row must pass `floor_coverage_pass` and `coop_member_value_pass`.

## 4) Weekly Operating Cadence

1. Monday: pipeline review (intros, onboarding status, risks).
2. Tuesday: migration and activation work block.
3. Wednesday: owner interviews and champion check-ins.
4. Thursday: product fixes for top onboarding friction.
5. Friday: KPI review + next-week commitments.

## 5) Growth Loop Rules (Non-Spam)

1. Warm intros only.
2. No purchased lists, no cold cadences.
3. Referral reward only after activation milestone.
4. Every referral must be explicit opt-in by the referred owner.
5. Complaint-triggered pause rule: if trust complaints rise, pause incentives.
